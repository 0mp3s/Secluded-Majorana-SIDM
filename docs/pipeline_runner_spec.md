# Pipeline Runner — מסמך אפיון

## מטרה
ממשק ויזואלי להרצת ה-pipeline המחקרי בלחיצה אחת, תוך שמירה על יכולת הרצה ידנית מהטרמינל ללא שום שינוי.

---

## ארכיטקטורה
| רכיב | טכנולוגיה |
|------|-----------|
| Backend | FastAPI (Python) |
| Frontend | HTML + vanilla JS |
| Streaming | WebSocket (לא polling) |
| הצגת תמונות | PNG inline בדפדפן |
| גישה | `http://localhost:8501` |
| Python לריצות | `py` (Python מערכת, יש numpy) |
| עצמאות | עובד בלי VS Code, מהטרמינל בלבד |

---

## מקור הנתונים
- הממשק קורא את `docs/execution_pipeline.csv` **בכל טעינה.**
- **File watcher** על הקובץ — שינוי ב-CSV = הממשק מתרענן אוטומטי ללא restart.
- עמודות בשימוש:

| עמודה | שימוש |
|-------|-------|
| `Order` | סדר הרצה |
| `Stage` | קיבוץ ויזואלי, אפשרות לדלג שלב שלם |
| `Script` | הפקודה: `py <Script>` |
| `Reads` | בדיקת קיום קלט לפני ריצה |
| `Writes` | סריקת output לאחר ריצה — הצגת PNG/CSV |
| `Dependencies` | תצוגת תלויות בממשק |
| `Paper Section` | תיוג בתצוגה |

---

## בחירת סקריפטים
- רשימת checkbox לכל סקריפט בנפרד, מקובצת לפי Stage.
- אפשרות לסמן / לבטל Stage שלם בבת אחת.
- "Run Selected" — מריץ רק את הסקריפטים המסומנים, לפי `Order`.

---

## קלט דינמי
- לקבצי קלט שיש מספר גרסאות (v31_, v34_...) — ברירת מחדל = **הקובץ עם תאריך שינוי הכי גבוה**.
- לפני ריצת כל סקריפט, המשתמש רואה איזה קובץ נבחר ויכול לבחור קובץ אחר.

---

## בדיקת קלט חסר
- לפני כל ריצה, הממשק בודק שכל הקבצים מ-`Reads` קיימים.
- קובץ חסר → **אזהרה צהובה**, לא חוסם (המשתמש מחליט אם להמשיך).
- תלויות שהן קוד (לא קבצים) — לא נבדקות.

---

## ריצה ו-Streaming
- stdout + stderr מוזרמים live דרך WebSocket.
- **אין חלונית קוד** בממשק.
- לאחר ריצה: סריקת תיקיות output לפי `Writes` → PNG מוצג inline, CSV מוצג כקישור.

---

## שליטה
| פעולה | תיאור |
|-------|-------|
| **Pause** | כפתור ידני — עוצר אחרי הסקריפט הנוכחי |
| **Resume** | ממשיך מהסקריפט הבא |
| **Stop** | עוצר לגמרי |
| **כשל (exit != 0)** | הודעה אדומה + עצירה אוטומטית, ממתין להחלטת המשתמש |
| **אזהרת ריצה ארוכה** | Stage 0 (v22_raw_scan) מקבל confirmation לפני ריצה |
